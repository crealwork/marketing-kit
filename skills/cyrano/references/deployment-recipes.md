# 배포 레시피

cyrano의 코어는 "이메일/캘린더 → 브리핑 → 전송"이다. **언제** 돌릴지는 호스트가 정한다.

## A. 아침 자동 브리핑

호스트 에이전트가 매일 아침 한 번, 그날의 외부 미팅을 브리핑한다.

**루틴 프롬프트(예):**
> "cyrano 스킬로 오늘 내 캘린더의 외부 미팅을 브리핑해줘. 각 미팅당 하나씩,
> 리서치→소스 인용된 브리핑→내 Slack DM으로."

**어디서 돌리나 — 이게 갈린다:**

- **클라우드 루틴**(CronCreate / `schedule` 스킬): 노트북 꺼져 있어도 뜬다. 대신
  로컬 자산(번들 insane-search 엔진, 로컬 API 키, 로컬 플러그인 MCP)에 **접근
  못 한다.** 캘린더는 계정 커넥터로 읽히고, 전송은 Slack/Telegram 웹훅 HTTP로 가능.
  리서치는 WebSearch/WebFetch 위주(차단 사이트 뚫기는 약해짐).
- **로컬 스케줄러**(Windows 작업 스케줄러 `schtasks`, macOS `launchd`): 풀 파워 —
  번들 insane-search, 로컬 키, 로컬 MCP 전부 사용. 대신 **기기가 켜져 있어야** 뜬다.
  로컬 시크릿을 쓰는 정기작업은 원격 루틴보다 로컬 스케줄러가 맞다
  ([[feedback_scheduled_task_local_secrets]], `schtasks` + `MSYS_NO_PATHCONV` 주의).

권장: 차단 사이트 리서치 품질이 중요하면 **로컬 스케줄러**, 신뢰성(항상 실행)이
중요하면 **클라우드 루틴**. 하이브리드도 가능(클라우드가 "오늘 외부 미팅 있음"만
알리고, 상세는 로컬에서).

**dedup:** 루틴이 재시도로 두 번 떠도 `engine check` → `--mark` 흐름이 중복 브리핑을
막는다. 리서치 **전** check, 전송 **후** mark.

## B. 온디맨드

> "cyrano로 john@acme.com 리서치해서 브리핑해줘"

filter 없이 이메일 하나를 바로 리서치 → 브리핑 → (return 모드면) 그 자리에서 보여주고,
채널 설정돼 있으면 전송.

## C. 캘린더 정규화 팁

호스트의 캘린더 도구가 주는 이벤트를 filter 입력 형태로 맞춘다:
```json
{"date":"YYYY-MM-DD","events":[
  {"id":"<event id>","title":"<제목>","start":"<ISO8601>",
   "attendees":["a@x.com","b@y.com"]}]}
```
- `id`는 dedup 키라 **안정적인 이벤트 ID**를 쓴다(없으면 제목으로 폴백).
- 참석자에 본인·회의실이 섞여도 filter가 걸러낸다.
- 종일 이벤트·취소된 이벤트는 호스트가 미리 제외하는 게 좋다.
